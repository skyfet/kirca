# Дизайн-система Kirca

Документ описывает принципы и строительные блоки интерфейса. Цель —
чтобы навигация, разделы и кнопки воспринимались как одно целое, а не как
набор разрозненных экранов. Любой новый экран должен собираться из этих
блоков, а не вводить собственные «магические числа».

## Принципы

1. **Один навигационный хребет.** Три верхнеуровневых раздела — «Комнаты»,
   «Друзья», «Профиль» — живут за постоянным нижним таб-баром
   (`GlassBottomBar`). Раньше они были спрятаны в иконки верхнего бара, а
   выход из аккаунта висел в одно касание на главном экране. Теперь верхний
   бар отвечает только за действия текущего раздела, а опасные действия
   (выход, удаление) собраны внизу профиля.

2. **Токены вместо чисел.** Отступы, радиусы, типографика и анимации берутся
   из `lib/theme/design.dart`. Никаких `fontSize: 15` или
   `borderRadius: 12` по месту — только `AppType`, `AppRadius`, `AppSpace`.

3. **Иерархия действий читается с первого взгляда.** Кнопка существует в трёх
   весах (`AppButton`): `primary` (акцентное свечение, одно главное действие
   на экране), `secondary` (нейтральные/отмена), `danger` (красная, для
   разрушительных). Пользователь не должен гадать, что важнее.

4. **Стекло — фон, контент — на первом плане.** Весь экран делит один
   liquid-glass слой (`AppScaffold`/`HomeShell`), поэтому панели, карточки и
   бар согласованно преломляют фон. Текст всегда `AppColors.onGlass*` с
   убыванием контраста для второстепенной информации.

5. **Спокойное движение.** Переходы короткие и единообразные (`AppMotion`),
   интерфейс не «суетится».

## Токены (`lib/theme/design.dart`)

| Группа     | Значения                                                        |
|------------|-----------------------------------------------------------------|
| `AppSpace` | сетка 4 pt: `xxs 2 · xs 4 · sm 8 · md 12 · lg 16 · xl 20 · xxl 24 · xxxl 32` |
| `AppRadius`| `sm 10` (чипы/бейджи) · `md 14` (карточки, инпуты, кнопки) · `lg 18` (диалоги, FAB, карточка логина) · `pill`. Плюс готовые `glassSm/Md/Lg` для glass-виджетов |
| `AppType`  | `display · title · heading · itemTitle · body · bodyMuted · caption · fine · button` |
| `AppMotion`| `fast 150ms · base 200ms · slow 300ms`, кривая `easeOutCubic`    |
| `AppSize`  | `fab 56 · iconButton 36 · buttonHeight 48 · compactButtonHeight 42 · avatar 44` |

Контраст текста задаётся палитрой из `app_theme.dart`:
`onGlass` (основной) → `onGlassMuted` → `onGlassDim` (подписи, таймстемпы).

## Компоненты (`lib/widgets/`)

- **`AppScaffold`** — единая обвязка страницы вместо повторявшегося
  `GlassPage → AdaptiveLiquidGlassLayer → Scaffold`. Режим `glass: true`
  (по умолчанию) для отдельных маршрутов; `glass: false` — для вкладок
  внутри `HomeShell`, которые наследуют общий стеклянный слой.

- **`AppButton`** (`.primary` / `.secondary` / `.danger`) — единственная
  кнопка приложения. Фиксированные радиус, высота, свечение и типографика;
  встроенные состояния `busy` (спиннер) и `icon`.

- **`TappableCard`** — `GlassCard` с ripple по тапу/долгому нажатию. Заменяет
  дублировавшийся паттерн `Stack(GlassCard, Positioned.fill(InkWell))` в
  списках комнат и друзей.

## Навигация

```
HomeShell (один GlassPage + стеклянный слой)
├── IndexedStack
│   ├── RoomsScreen      (вкладка, AppScaffold glass:false)
│   ├── FriendsScreen    (вкладка, табы внутри: Друзья/Запросы/Приглашения)
│   └── ProfileScreen    (вкладка; внизу — Выйти / Выйти со всех / Удалить)
└── GlassBottomBar (Комнаты · Друзья · Профиль), бейдж на «Друзья» = запросы+приглашения
```

Глубинные экраны (`ChatScreen`, `ArchivedRoomsScreen`, `BlockedUsersScreen`,
`MembersScreen` …) по-прежнему открываются как отдельные маршруты через
`AppScaffold` с обычной кнопкой «назад».

## Как добавлять экран

1. Оберни в `AppScaffold` (или сделай вкладкой `HomeShell` c `glass:false`).
2. Заголовок — `GlassAppBar(title: Text('…', style: AppType.title))`.
3. Отступы/радиусы — только из `AppSpace`/`AppRadius`.
4. Кнопки — `AppButton` нужного веса; ровно одна `primary` на экран.
5. Кликабельные строки списков — `TappableCard`.
6. Текст — стили из `AppType`, контраст — палитрой `onGlass*`.
